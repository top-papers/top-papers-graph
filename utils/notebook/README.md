# notebook/

Исходный Colab-блокнот Task 3 для эксперта-создателя набора.

## Файл
- `expert_task3_yaml_bundle_for_ab_test.ipynb` — блокнот, который собирает офлайн-форму
  создателя набора, шаблон манифеста и сопутствующие артефакты.

## Зачем он здесь
Это **источник истины по схеме** манифеста. Все поля, enum-значения и правило
готовности кейса («Готов к экспорту») взяты именно отсюда (ячейка-сборщик формы,
cell 8), а не из текста задания.

Ключевые элементы схемы, извлечённые из блокнота:
- структура манифеста: `{ schema_version, experiment_meta, cases[] }`;
- `experiment_meta`: `topic, submission_id, creator_id, cutoff_year, review_goal`;
- правило готовности (`requiredMissing`): непустые `paper_title`, `creator_prompt`,
  `creator_rationale`, `review_focus` (≥1) и хотя бы одно поле `match.*`.

## Важное замечание
В этой копии блокнота ячейка-сборщик формы (cell 8) намеренно отредактирована
(литеральные `...`, заглушка `def build_authoring_bundle(...)`) и **не исполняется
целиком**. При этом сам HTML/JS-шаблон формы и все константы схемы в ячейке
присутствуют.

Поэтому веб-форма в [`../web_form/`](../web_form/) **реконструирована из подлинного
HTML-шаблона той же ячейки** скриптом [`../scripts/build.py`](../scripts/) —
чтобы манифест реально загружался и проходил статусы готовности. Полноценную форму
также можно получить, запустив неотредактированный блокнот в Colab.
