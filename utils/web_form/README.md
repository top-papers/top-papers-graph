# web_form/

GUI-форма создателя набора Task 3. **Пуста по умолчанию**; кейсы приходят из
[`../config.yaml`](../config.yaml) после `python3 scripts/build.py`. Все параметры
(метаданные эксперимента и кейсы) заполняются через браузер. Работает в двух режимах:
как веб-сервер и как офлайн-файл.

## Файлы
- `serve.py` — веб-сервер (только стандартная библиотека Python, без зависимостей).
- `task3_ab_creator_offline_form_ru.html` — сама страница (HTML+JS). Открывается и
  локально как файл, и через `serve.py`.

## Запуск как веб-сервер (рекомендуется)
```bash
python3 web_form/serve.py            # http://localhost:9000
python3 web_form/serve.py --port 9001  # если порт 9000 занят
```
Откройте `http://localhost:9000`:
1. Страница сама подтягивает текущий манифест с диска (`GET /api/manifest`).
2. Заполняйте метаданные и кейсы; статус каждой карточки пересчитывается на лету.
3. **«Сохранить на сервер»** → манифест пишется прямо в корневой
   [`../task3_ab_case_manifest.filled.json`](../task3_ab_case_manifest.filled.json)
   (`POST /api/manifest`; перед перезаписью создаётся `.bak`).

Маршруты сервера: `GET /` (форма), `GET /api/manifest` (читает с диска),
`POST /api/manifest` (пишет на диск).

## Запуск как офлайн-файл
Просто откройте `task3_ab_creator_offline_form_ru.html` в браузере. Кнопка
**«Скачать заполненный JSON»** сохранит итог в Загрузки, а **«Загрузить черновик
JSON»** подгрузит ранее сохранённый файл. Кнопка «Сохранить на сервер» в офлайн-режиме
сообщит, что сервер недоступен — это нормально.

## Происхождение
Форма **не написана с нуля**, а реконструирована из подлинного HTML-шаблона
ячейки-сборщика (cell 8) блокнота [`../notebook/`](../notebook/) скриптом
[`../scripts/build.py`](../scripts/) — поэтому её поведение и правило готовности кейса
в точности совпадают с оригиналом. Билдер также убирает авто-создание стартовых кейсов
(форма открывается пустой).
