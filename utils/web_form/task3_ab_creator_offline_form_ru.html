Task 3 — форма создателя набора A/B

Эту форму заполняет эксперт-создатель. Её задача — подготовить диагностические кейсы, на которых различия между baseline VLM и SFT/DPO VLM будут видны именно там, где это важно для Task 3: в multimodal evidence extraction, evidence linkage и temporal grounding.

Как пользоваться формой
1) заполните метаданные эксперимента; 2) добавляйте и убирайте кейсы по нужным стратам кнопками + и -; 3) при необходимости скачайте черновик и продолжите позже через загрузку JSON; 4) для каждого кейса кратко сформулируйте, что должен оценивать участник A/B; 5) при необходимости раскройте технический блок сопоставления; 6) скачайте заполненный JSON или перейдите в Google Форму для отправки данных.
Открыть Google Форму для отправки данных
Работа с черновиком
Нажмите «Скачать черновик JSON», если хотите продолжить позже или передать незавершённый вариант коллеге. Чтобы восстановить работу, используйте «Загрузить черновик JSON». Для быстрой отправки данных в веб-канал используйте кнопку «Открыть Google Форму для отправки данных».

1. Метаданные эксперимента

Сначала зафиксируйте тему, submission_id и цель проверки. Эти поля попадут в итоговый manifest.

2. Кейсы для набора

Рекомендуемый баланс: больше multimodal_hard и temporal_hard, меньше easy_control.

Используйте кнопки + и - для быстрого добавления и удаления карточек кейсов прямо в офлайн-форме.
