# Роли и ответственность (SciReason / “Научный Экзокортекс”)

Ниже — “карта людей”, чтобы **200–300 участников не превратились в хаос**.

## 1) Управление программой (Core Team)

### Program Lead (Руководитель программы)
**Зачем нужен:** держит цель, сроки и правила игры.  
**Делает:**
- утверждает темы потоков и критерии “готово”,
- назначает Scientific Leads по доменам,
- решает конфликтные вопросы и приоритеты,
- выпускает еженедельный статус (1 страница).

### Scientific Lead (Научный лидер домена: Bio/Chem/Phys/…)
**Зачем нужен:** научная строгость и единые стандарты.  
**Делает:**
- формирует “тематики-спринты” для домена,
- утверждает рубрики (что такое “хороший” граф/траектория/рецензия),
- финально принимает “золотые стандарты”.

### Tech Lead (Архитектор/ведущий разработчик)
**Зачем нужен:** не дать проекту “разъехаться” по технологиям.  
**Делает:**
- определяет архитектуру и интерфейсы (API/форматы),
- принимает изменения в инфраструктуре/пайплайнах,
- следит за стабильной сборкой и тестами.

### Data Steward (Стюард данных)
**Зачем нужен:** чтобы артефакты от людей были пригодны для обучения/оценки.  
**Делает:**
- контролирует форматы (`docs/artifacts.md`),
- делает “валидацию PR” на данные,
- ведёт словарь доменных терминов (glossary) и правила именования.

### Community / Ops (Комьюнити-менеджер / координатор)
**Зачем нужен:** 300 людей нужно беречь: онбординг, FAQ, геймификация.  
**Делает:**
- собирает потоки/отряды (squads),
- организует регулярные синки,
- ведёт лидерборды и благодарности.

---

## 2) Squads (кросс-функциональные отряды)

**Состав (рекомендация):** 1 ML/Backend инженер + 1 научный модератор + 3–7 экспертов.  
**Задача отряда:** за 2 недели “закрыть” одну микротему: собрать статьи → построить граф → проверить → сделать 2–5 гипотез → получить рецензии.

---

## 3) Эксперты (Domain Experts)

Эксперты **не занимаются разметкой ради разметки**. Их вклад — *supervision of reasoning*:

1) **Reasoning Trajectories** (золотой стандарт рассуждений)  
2) **Graph Verification** (подтверждение/резка связей)  
3) **Hypothesis Red Teaming** (жёсткая рецензия гипотез)

---

## 4) Разработчики и ML-инженеры

### ML Engineer
- экстракция триплетов, графовые веса,
- промпты/структурный вывод,
- оценка качества (evals), регрессии.

### Backend Engineer
- API и хранение артефактов,
- пайплайн ingestion: pdf → text → chunks → embeddings → graph.

### DevOps / Platform
- docker-compose / k8s,
- мониторинг (опционально), секреты, доступы.

### QA / Release
- тестирование пайплайнов,
- релизы, чтобы эксперты всегда работали в стабильном UI.

---

## 5) “Кто что делает” (супер-коротко)

- **Эксперт** → создаёт файлы в `data/experts/...` (по шаблонам) + ревьюит результаты системы.
- **Инженер** → делает так, чтобы эти файлы улучшали систему: промпты, метрики, веса, модели.
- **Лидеры** → следят за стандартами, темами, качеством и итоговыми выводами.
