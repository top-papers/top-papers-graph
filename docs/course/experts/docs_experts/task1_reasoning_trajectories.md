# Task 1 — Reasoning Trajectories (Золотой стандарт рассуждений)

## Цель
Научить систему строить логические цепочки, приводящие к **проверяемому** выводу на основе источников.

## Выходной артефакт
1 YAML файл: **шаги 1…N** (без верхнего ограничения) + опциональные связи `edges` между шагами.

## Обязательные поля каждого шага (artifact v2)
- `claim` — утверждение/факт
- `sources` — список источников (можно 1, можно много), каждый элемент:
  - `type`: `text` / `image` / `table`
  - `source`: DOI/arXiv/OpenAlex/URL/…
  - `snippet_or_summary`: цитата / краткая выжимка / описание фигуры/таблицы
  - `page` и `locator` — если применимо (особенно для figure/table)
- `conditions` — условия применимости (домен‑зависимые поля могут быть обязательными)
- `inference` — почему следующий шаг логичен
- `next_question` — вопрос/следующий шаг рассуждения

## Связи между шагами (edges)
`edges` — это **ориентированный** граф в виде списка пар:
```yaml
edges:
  - [1, 2]
  - [1, 3]
```
Интерпретация: шаг 1 ведёт к шагам 2 и 3.

## Домен
Поле `domain` хранит **Wikidata QID** (например `Q336`). Валидация домен‑специфичных `conditions` делается по маппингу `wikidata_qid` в `configs/domains/*.yaml`.

## Шаблон
См. `data/experts/trajectories/_template.yaml` (и пример в `examples/.../data/experts/trajectories/`).
