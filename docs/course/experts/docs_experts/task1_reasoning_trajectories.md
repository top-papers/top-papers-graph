# Task 1 — Reasoning Trajectories (Золотой стандарт рассуждений)

## Цель
Научить систему строить логические цепочки, приводящие к **проверяемому** выводу.

## Выходной артефакт
1 YAML файл: 6–12 шагов.

## Обязательные поля каждого шага (temporal + multimodal)
- claim (факт)
- evidence (текст/фигура/таблица + страница + номер)
- conditions (T, SOC окно, chemistry, формат, aging если есть)
- inference (почему следующий шаг логичен)
- next_question

## Главное правило
Если evidence в фигуре — кратко опишите оси и тренд (чтобы проверяющий понял, что вы действительно читали график).

## Шаблон
См. `templates/trajectory_template.yaml`.
