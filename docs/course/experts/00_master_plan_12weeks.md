# top-papers-graph — план-график на 12 недель (temporal graphs + multimodal VL)

**Контекст:** курс/проект по научным агентам для *любой* научной тематики (домен выбираете вы).  
**Фокус экспертов:** 3 ключевые задачи (Trajectories, Graph Verification, Hypothesis Red Teaming) + обновления под **темпоральность** и **мультимодальность**.  
**Формат управления:** 1 общий синк в неделю + асинхронная работа через PR/issue. (Доменный валидатор/эксперимент — опционально.)

> Примечание: в качестве **примера** доменного валидатора в репозитории есть `examples/battery_fastcharge/` (PyBaMM).

## Роли (минимальный набор)
- **Program Lead:** держит цели/метрики, принимает решения по scope.
- **Tech Lead:** отвечает за пайплайн (ingest → MMKG/TKG → GraphRAG → debate → (domain validation)).
- **Squad Lead (1 на домен/подтему):** распределяет задачи, следит за DoD.
- **Эксперты:** выполняют 3 задачи разметки/ревью.
- **QC-редактор (1–2 человека):** выборочная проверка, ведение чек-листов, калибровка.

## Definition of Done (DoD) для экспертов (коротко)
- **Trajectories:** 1 YAML = 6–12 шагов, каждый шаг с evidence (цитата/страница/фигура), и указанием времени/условий.
- **Graph review:** 1 JSON = 30–80 утверждений (assertions) с вердиктом accepted/rejected/needs_time_fix + причины + ссылки.
- **Hypothesis review:** 1 JSON = оценка гипотезы + критические замечания + проверяемость + временной скоуп.

## Недельные встречи
- 60 минут. Повестки: см. `agendas/weekXX_agenda.md`.
- Принцип: таймбоксинг + Parking Lot (чтобы встреча не “утекала”).

## Календарь на 12 недель (по неделям)

> Даты ориентировочные (старт с ближайшего понедельника): **2026-01-26**.

### Неделя 1 — Онбординг + калибровка экспертов (Calibration)
**Цель:** выровнять качество разметки и термины, собрать “калибровочный набор”.
- Эксперты:
  - Размечают **один и тот же** “калибровочный” paper (по профилям fast-charge).
  - Делают: 1 mini-trajectory (3–5 шагов) + 1 mini-graph review (10–20 assertions) + 1 mini-hypothesis review.
- Dev/ML:
  - Поднимают инфраструктуру (Neo4j/Qdrant/MM ingest).
  - Делают экспорт страниц/фигур (multimodal) и год/условия (temporal).
**Deliverables:**
- 1 калибровочный пакет артефактов на всех.
- Список “спорных мест” + обновлённый глоссарий.

### Неделя 2 — Набор корпуса (v0) + первая “золотая” траектория
**Цель:** стартовый корпус + первый эталон рассуждения.
- Эксперты:
  - 2 trajectories (полные, 6–12 шагов) по ключевым работам.
- Dev/ML:
  - Авто-извлечение утверждений + построение TKG/MMKG v0.
**Deliverables:** Trajectories v0, граф v0, список ошибок извлечения.

### Неделя 3 — Graph Verification v1 (основная чистка) + привязка к фигурам
**Цель:** убрать галлюцинации связей и добавить multimodal evidence.
- Эксперты:
  - 3 graph reviews (по 30–80 assertions).
  - Отмечают, какие связи опираются на **график/таблицу** (страница/номер фигуры).
- Dev/ML:
  - Учитывают ревью: ребра получают веса, rejected удаляются/понижаются.
**Deliverables:** граф v1 (чистый), MM evidence-линки.

### Неделя 4 — Debates v1 + Red Teaming v1
**Цель:** первый полноценный цикл “hypothesis → critique → revise”.
- Эксперты:
  - 5 hypothesis reviews (по чек-листу).
- Dev/ML:
  - Включают чек-листы экспертов в промпт Скептика.
**Deliverables:** 5 рецензий, 2–3 улучшенные гипотезы.

### Неделя 5 — Domain validation v1 (репликация литературы)
**Цель:** воспроизвести 2–3 профиля зарядки из литературы в симуляции как baseline.
- Эксперты:
  - Проверяют корректность интерпретации профилей (SOC окна, температуры, условия).
- Dev/ML:
  - Прогоны domain validation, генерация графиков/таблиц.
**Deliverables:** таблица baseline-результатов + репорт reproducibility.

### Неделя 6 — Temporal sanity v1 (борьба с “темпоральными галлюцинациями”)
**Цель:** научить граф разделять факты по времени/условиям.
- Эксперты:
  - 2 graph reviews, но с фокусом: **временные метки**, условия эксперимента, “что где применимо”.
- Dev/ML:
  - Темпоральный слой: assertions с `time_interval` и `conditions`.
**Deliverables:** TKG v1, отчёт по конфликтам.

### Неделя 7 — Debates v2 (с ограничениями: время/условия) + Red Teaming v2
**Цель:** гипотезы должны быть корректно “заземлены” по времени и условиям (T, SOC, chemistry).
- Эксперты:
  - 6 hypothesis reviews (с обязательным временным скоупом).
- Dev/ML:
  - Добавляют query decomposition по времени/условиям.
**Deliverables:** лучшие 1–2 гипотезы недели + список “fail modes”.

### Неделя 8 — MMKG v2 (таблицы/графики как первоклассные сущности)
**Цель:** улучшить извлечение из графиков/таблиц и связку “текст ↔ визуал”.
- Эксперты:
  - 10 “figure cards” (карточки: что на графике, оси, вывод, условия).
- Dev/ML:
  - VL-подписи страниц (если доступно) + OpenCLIP индекс.
**Deliverables:** MMKG v2, улучшение retrieval по визуальным данным.

### Неделя 9 — Экспериментальные гипотезы: 2 кандидата + Domain validation v2
**Цель:** выбрать 2 кандидата гипотез и проверить их в domain validation.
- Эксперты:
  - 2 “go/no-go” ревью (жёсткие) на 2 гипотезы.
- Dev/ML:
  - Sweep параметров, устойчивость, ограничения.
**Deliverables:** 2 verified candidates (или честный negative result).

### Неделя 10 — Writing sprint v1 (черновик статьи)
**Цель:** собрать paper skeleton: novelty → verification → limitations → disclosure.
- Эксперты:
  - Редактура “limitations” и “domain correctness”.
- Dev/ML:
  - Вставляют фигуры/таблицы, формируют supplementary.
**Deliverables:** черновик v1 + артефакты.

### Неделя 11 — Writing sprint v2 (rigor + reproducibility)
**Цель:** довести воспроизводимость и “audit readiness”.
- Эксперты:
  - Финальная проверка ссылок/цитат (spot-check 15–20 утверждений).
- Dev/ML:
  - `make reproduce`, фиксация версий, seeds.
**Deliverables:** paper v2, reproducibility report.

### Неделя 12 — Предсабмит аудит + финализация
**Цель:** “не за что отклонять”: формальности, disclosure, анонимизация.
- Эксперты:
  - Финальный review “как рецензент”.
- Dev/ML:
  - Pre-submit audit, clean PDF, supplementary.
**Deliverables:** финальный пакет статьи.

## Количественные цели для экспертов (рекомендуемые)
За 12 недель на 1 активную эксперт-группу (5–10 человек):
- **Trajectories:** 6–10 полных
- **Graph reviews:** 10–15 (по 30–80 assertions)
- **Hypothesis reviews:** 15–25
- **Figure cards:** 10–20
