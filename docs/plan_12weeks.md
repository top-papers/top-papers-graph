# 12-недельный план (универсальный)

Этот файл — операционный план для запуска *любого* научного домена в top-papers-graph.
Роль “валидации гипотезы” зависит от домена: это может быть симулятор, лабораторный протокол, офлайн‑эксперимент, репликация на датасете и т.п.

## Недели 1–2: Baseline + reasoning trajectories
- Dev:
  - поднять инфраструктуру (docker compose),
  - настроить LLM/эмбеддинги,
  - сделать baseline пайплайна (ingest → нормализация PaperMetadata → KG) на 10–20 статьях.
- Experts:
  - выбрать 2–3 под‑темы,
  - заполнить 2–3 reasoning trajectories (E1) с обязательным evidence.

## Недели 3–5: GraphRAG + граф‑ревью
- Dev:
  - стабильный ingest (PDF + метаданные),
  - извлечение сущностей/триплетов → KG,
  - GraphRAG запросы под research question.
- Experts:
  - E2: проверка графов, уточнение условий применимости, фиксы темпоральности.

## Недели 6–8: Дебаты агентов + чек‑лист Скептика
- Dev:
  - внедрить чек‑лист Скептика (configs/checklists/*),
  - сохранять логи дебатов и артефакты.
- Experts:
  - уточнить/расширить доменный чек‑лист,
  - начать E3 (рецензии гипотез).

## Недели 9–10: Валидация гипотезы + абляции
- Dev:
  - подключить доменный “валидатор” (эксперимент/симулятор/репликация),
  - ablation: без графа, без multimodal, без temporal fixes и т.д.
- Experts:
  - оценка соответствия гипотезы источникам + риски/ограничения.

## Недели 11–12: Упаковка результата
- отчёт (что подтверждено / что опровергнуто / где неопределённость),
- reproducibility pack (список источников, конфиги, команды запуска),
- план next steps (какие данные/эксперименты закрывают неопределённость).
