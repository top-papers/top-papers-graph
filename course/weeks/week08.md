## Теория (простыми словами)

- Идея “векторизации” вершин: близкие узлы → близкие векторы.
- Простой подход: спектральное вложение (eigenvectors) или random-walk идеи (Node2Vec).
- Как это использовать для ранжирования кандидатов ребер.

## Практика в контексте проекта

- Реализовать простой spectral embedding (numpy) для графа и посчитать cosine similarity.
- Сравнить top кандидатов по similarity и по Adamic–Adar.
- Добавить простой ансамбль скоринга (0.5*AA + 0.5*cosine).
