# Чеклист эксперта перед A/B тестом

## Что эксперт должен получить
- offline HTML для blind review;
- public manifest без owner key;
- этот пакет шаблонов;
- короткое описание цели: сравниваются только VLM-слои, а не разные retrieval/LLM пайплайны.

## Что эксперт должен проверить до старта
- blind key отсутствует;
- в наборе достаточно multimodal hard примеров;
- пары сопоставлены по одной и той же semantic unit, а не только по рангу;
- видны evidence snippets, time scope и source locator.

## Что эксперт должен подготовить сам
- reviewer_id / имя для финального JSON;
- 60–90 минут непрерывного времени на 20–30 пар;
- правило: оценивать сначала evidence/time correctness, а не красоту формулировки.

## На что смотреть в первую очередь
1. Не пропущен ли факт из figure/table/formula.
2. Правильно ли привязана страница / figure / table.
3. Нет ли визуальной галлюцинации.
4. Верно ли отражён temporal scope.
5. Не сглажена ли разница только за счёт более гладкого prose.
