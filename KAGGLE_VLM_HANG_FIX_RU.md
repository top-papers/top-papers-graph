# Исправление возможного подвисания VLM на Kaggle

Что исправлено:
- добавлен жёсткий тайм-аут ожидания ответа от local VLM worker (`LOCAL_VLM_REQUEST_TIMEOUT_SECONDS`, по умолчанию 120 сек);
- если worker зависает на загрузке модели / генерации / скачивании весов, процесс страницы больше не висит бесконечно; worker закрывается, VLM отключается для оставшихся страниц, а pipeline продолжает работу;
- для Kaggle добавлены безопасные defaults: `PDF_RENDER_DPI=110`, ограничение `VLM_MAX_PIXELS`, единый `HF_HOME/HF_HUB_CACHE` в `/kaggle/working/.hf`.

Почему это важно:
- в логе были повторяющиеся page-level вызовы VLM и длительные интервалы перед warning, что похоже на зависание worker на первой генерации или загрузке модели;
- теперь такой сценарий заканчивается fail-fast, а не бесконечным ожиданием.
